# Annotation Safety Fix

## Проблема

Аннотации могли выходить за пределы изображения:
1. Badge вылезал за нижнюю границу (bottom: -28px)
2. Нейронка могла передать координаты > 100 или < 0
3. Нейронка могла передать x2 < x1 или y2 < y1
4. Всё ломалось и выглядело криво

## Решение

### 1. Валидация на стороне skill (первый уровень защиты)

В `lib/skills/built-in/image-analyser/skill.ts`:

```typescript
// Клампим координаты к 0-100
const x1 = Math.max(0, Math.min(100, Number(ann.x1_pct)));
const y1 = Math.max(0, Math.min(100, Number(ann.y1_pct)));
const x2 = Math.max(0, Math.min(100, Number(ann.x2_pct)));
const y2 = Math.max(0, Math.min(100, Number(ann.y2_pct)));

// Проверяем что x2 > x1 и y2 > y1
if (x2 <= x1 || y2 <= y1) {
  return { error: 'Invalid coordinates after clamping' };
}

// Обновляем на клампнутые значения
ann.x1_pct = x1;
ann.y1_pct = y1;
ann.x2_pct = x2;
ann.y2_pct = y2;
```

**Что делает:**
- Автоматически исправляет координаты если нейронка ошиблась
- Если x1=105, станет x1=100
- Если x1=-10, станет x1=0
- Если после клампинга x2 <= x1, возвращает ошибку

### 2. Валидация на стороне UI (второй уровень защиты)

В `components/AnnotationOverlay.tsx`:

```typescript
// Клампим координаты к 0-100 для безопасности
const x1 = Math.max(0, Math.min(100, annotation.x1_pct));
const y1 = Math.max(0, Math.min(100, annotation.y1_pct));
const x2 = Math.max(0, Math.min(100, annotation.x2_pct));
const y2 = Math.max(0, Math.min(100, annotation.y2_pct));

// Проверяем что x2 > x1 и y2 > y1
if (x2 <= x1 || y2 <= y1) {
  console.warn('Invalid annotation coordinates:', annotation);
  return null; // Не рендерим эту аннотацию
}
```

**Что делает:**
- Дополнительная защита на случай если skill пропустил
- Просто не рендерит невалидные аннотации
- Логирует в консоль для дебага

### 3. Badge внутри изображения

Было:
```css
bottom: -28px; /* Вылезал за пределы */
```

Стало:
```css
bottom: 4px; /* Внутри рамки */
```

**Что изменилось:**
- Badge теперь ВНУТРИ аннотации, а не снаружи
- Не вылезает за границы изображения
- Выглядит аккуратно

## Примеры

### ✅ Валидные координаты
```json
{
  "x1_pct": 10,
  "y1_pct": 20,
  "x2_pct": 50,
  "y2_pct": 60
}
```
Результат: рендерится нормально

### ✅ Координаты за пределами (автоисправление)
```json
{
  "x1_pct": -10,  // станет 0
  "y1_pct": 20,
  "x2_pct": 150,  // станет 100
  "y2_pct": 60
}
```
Результат: клампится к 0-100, рендерится нормально

### ❌ Невалидные координаты (после клампинга)
```json
{
  "x1_pct": 80,
  "y1_pct": 20,
  "x2_pct": 70,  // x2 < x1
  "y2_pct": 60
}
```
Результат: ошибка от skill "x2 must be > x1"

### ❌ Невалидные координаты (экстремальный случай)
```json
{
  "x1_pct": 150,  // станет 100
  "y1_pct": 20,
  "x2_pct": 200,  // станет 100
  "y2_pct": 60
}
```
Результат: после клампинга x1=100, x2=100 → x2 <= x1 → ошибка

## Тестирование

Попробуй попросить нейронку:
1. "Обведи кота" — должно работать
2. "Обведи что-то за пределами фото" — должна получить ошибку или автоисправление
3. "Обведи всё фото" (0-100) — должно работать
4. Несколько аннотаций одновременно — все должны быть внутри

## Итог

Теперь система защищена на двух уровнях:
1. **Skill** — клампит и валидирует перед созданием артефакта
2. **UI** — клампит и валидирует перед рендерингом

Badge не вылезает за пределы, всё выглядит аккуратно! 🎉
