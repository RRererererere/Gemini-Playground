# 🔥 GeminiBridge — Тестирование AI-интерактивных сайтов

## Что было исправлено

### Проблема
GeminiBridge не работал из-за того, что `bridgeScript` генерировался один раз при первом рендере компонента, и `previewMode` внутри скрипта не обновлялся при переключении режимов. Это приводило к тому, что:
1. GeminiBridge инициализировался с неправильным режимом
2. При переключении в режим "AI App" скрипт не перезагружался
3. Данные с сайта не отправлялись в чат

### Решение
1. **Преобразовали `bridgeScript` в функцию `getBridgeScript()`** — теперь скрипт генерируется динамически с актуальным `previewMode`
2. **Добавили useCallback** — мемоизация функции с зависимостью от `previewMode`
3. **Добавили useEffect для перезагрузки iframe** — при смене режима iframe полностью перезагружается с новым скриптом
4. **Добавили console.log** — для отладки можно видеть в консоли браузера, что GeminiBridge инициализирован
5. **Улучшили UX** — добавили автоблокировку кнопки отправки и красивое отображение данных в чате

## Новые улучшения

### 1. Автоблокировка кнопки отправки
AI теперь автоматически генерирует код с блокировкой кнопки после отправки:
```javascript
submitBtn.disabled = true;
submitBtn.textContent = 'Отправлено...';
```
Это предотвращает спам и улучшает UX.

### 2. Красивое отображение данных в чате
Вместо уродливого JSON текста, данные от сайта теперь отображаются в стильном компоненте:
- Иконка и цвет зависят от типа события (форма, калькулятор, опрос и т.д.)
- Сворачиваемый блок с данными
- Подсветка синтаксиса JSON
- Адаптивный дизайн

## Как протестировать

### Шаг 1: Создайте AI-интерактивный сайт

Попросите Gemini создать сайт с формой:

```
Создай форму обратной связи с полями: имя, email, сообщение. 
Тип сайта: ai_interactive. 
При отправке формы данные должны отправляться в чат через GeminiBridge.
```

### Шаг 2: Проверьте режим

После генерации сайта:
1. Откройте DevTools (F12) → вкладка Console
2. Вы должны увидеть: `[GeminiBridge] Initialized successfully`
3. Внизу preview должна быть кнопка переключения режимов
4. Убедитесь что выбран режим **"AI App"** (с иконкой молнии ⚡)

### Шаг 3: Отправьте данные

1. Заполните форму на сайте
2. Нажмите Submit
3. В консоли должно появиться: `[GeminiBridge] Sending to AI: form_submission {...}`
4. В чате должно появиться новое сообщение от пользователя с данными формы:
   ```
   [🌐 SITE DATA] form_submission
   ```json
   {
     "name": "...",
     "email": "...",
     "message": "..."
   }
   ```
   ```
5. Gemini должен проанализировать данные и ответить

### Шаг 4: Проверьте разные режимы

- **Interact** — форма работает как обычно, данные НЕ отправляются в AI
- **AI App** — данные отправляются в AI через GeminiBridge
- **Inspect** (только для static сайтов) — клик на элемент отправляет информацию в чат

## API GeminiBridge

### Отправка данных в AI
```javascript
window.GeminiBridge.send(eventType, data, format);
```

**Параметры:**
- `eventType` (string) — тип события (например, "form_submission", "calculation", "survey_complete")
- `data` (object) — данные для отправки
- `format` (optional) — "auto" (по умолчанию), "silent", "show"

**Пример:**
```javascript
window.GeminiBridge.send('user_feedback', {
  rating: 5,
  comment: 'Great product!',
  timestamp: Date.now()
});
```

### Показать состояние загрузки
```javascript
window.GeminiBridge.setLoading(true);  // Показать индикатор
window.GeminiBridge.setLoading(false); // Скрыть индикатор
```

### Получить ответ от AI (опционально)
```javascript
window.GeminiBridge.onResponse((type, payload) => {
  console.log('AI ответил:', type, payload);
  // Обработать ответ
});
```

## Отладка

### Проверка инициализации
Откройте консоль браузера (F12) и проверьте:
```javascript
window.GeminiBridge
// Должен вернуть объект с методами: send, onResponse, setLoading
```

### Проверка режима
В консоли iframe должно быть:
```
[GeminiBridge] Initialized successfully
[LivePreview] Mode: ai-app
```

### Проверка отправки
При вызове `GeminiBridge.send()` в консоли должно появиться:
```
[GeminiBridge] Sending to AI: <eventType> <data>
```

## Известные ограничения

1. **GeminiBridge работает только в режиме "AI App"** — в режимах "Interact" и "Inspect" мост не активен
2. **Ответы AI идут в чат, не обратно на сайт** — это односторонняя коммуникация (сайт → AI → чат)
3. **Для двусторонней коммуникации** используйте `onResponse()` (пока не реализовано полностью)

## Следующие шаги

Если хотите добавить двустороннюю коммуникацию (AI → сайт):
1. Раскомментируйте код в `LivePreviewPanel.tsx` (метод `sendAIResponseToSite`)
2. Добавьте логику в `page.tsx` для отправки ответов AI обратно на сайт
3. Используйте `window.GeminiBridge.onResponse()` на сайте для обработки ответов
