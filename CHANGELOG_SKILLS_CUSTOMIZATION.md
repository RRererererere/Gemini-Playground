# Changelog: Skills Customization System

## Версия 1.0.0 - Система кастомизации скилов

### 🎉 Новые возможности

#### Полная кастомизация скилов
- **Кастомные описания** — изменяйте описание скилла в маркете
- **Кастомные системные промпты** — переопределяйте инструкции для AI
- **Кастомные описания инструментов** — настраивайте каждый tool отдельно

#### UI для настройки
- Новая вкладка "Промпт" в настройках скилла
- Новая вкладка "Инструменты" для настройки tool descriptions
- Показ дефолтных значений для сравнения
- Кнопка "Сбросить к умолчаниям"

#### Экспорт/Импорт
- Экспорт всех настроек скилов в JSON
- Импорт настроек из файла
- Перенос конфигурации между устройствами
- Шаринг настроек с другими пользователями

### 🔧 Технические изменения

#### Типы (`lib/skills/types.ts`)
```typescript
// Новый интерфейс для кастомизации
interface SkillCustomization {
  customDescription?: string;
  customSystemPrompt?: string;
  customToolDescriptions?: Record<string, string>;
}

// Обновлён InstalledSkillRecord
interface InstalledSkillRecord {
  // ... существующие поля
  customization?: SkillCustomization;
}
```

#### Registry (`lib/skills/registry.ts`)
Новые функции:
- `getSkillCustomization(id)` — получить кастомизацию
- `saveSkillCustomization(id, customization)` — сохранить
- `resetSkillCustomization(id)` — сбросить к умолчаниям
- `exportSkillsSettings()` — экспорт в JSON
- `importSkillsSettings(data)` — импорт из JSON

#### Executor (`lib/skills/executor.ts`)
- `buildSkillsSystemPrompt()` — использует кастомный промпт если есть
- `collectSkillTools()` — применяет кастомные описания инструментов

#### Компоненты
- `SkillsMarket.tsx` — обновлён ConfigModal с новыми вкладками
- Добавлены кнопки экспорта/импорта в header маркета
- Улучшен UX настройки скилов

### 🚀 Как использовать

1. Откройте Маркет скиллов (✨ Скиллы)
2. Выберите скилл → нажмите ⚙️ Настройки
3. Перейдите на вкладку "Промпт" или "Инструменты"
4. Настройте под свои нужды
5. Нажмите "Сохранить кастомизацию"

Для экспорта/импорта используйте кнопки 📥/📤 в header маркета.

### 🎨 Примеры кастомизации

#### Calculator - строгий режим
```
Системный промпт:
КРИТИЧЕСКИ ВАЖНО: ВСЕГДА используй calculate для ЛЮБЫХ вычислений.
НИКОГДА не считай в уме. Это обязательное требование.

Tool description (calculate):
ОБЯЗАТЕЛЬНЫЙ инструмент для ВСЕХ вычислений без исключений.
Вызывай ПЕРЕД тем как дать любой числовой ответ.
```

#### Notes - проактивный режим
```
Системный промпт:
ВАЖНО: Активно предлагай сохранять в заметки важные факты, 
задачи, идеи и ссылки. Спрашивай: "Сохранить это в заметки?"
```

### 🐛 Исправленные проблемы

- AI иногда игнорировал дефолтные инструкции скилов
- Не было способа настроить поведение скилов без изменения кода
- Невозможно было поделиться настройками между устройствами

### ⚡ Производительность

- Кастомизация загружается из localStorage (мгновенно)
- Не влияет на скорость работы скилов
- Минимальный overhead при сборке промптов

### 🔐 Безопасность

- Все данные хранятся локально (localStorage)
- Нет отправки настроек на сервер
- Экспорт/импорт полностью контролируется пользователем
