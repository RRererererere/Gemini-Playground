# Annotation Markers Feature - Визуальные ссылки на области изображений

## Статус: ✅ ГОТОВО К ТЕСТИРОВАНИЮ

Реализована система визуальных маркеров (@кольца) для ссылок на аннотации изображений с интерактивным просмотром.

---

## Как работает

### 1. Клик на аннотацию
Когда пользователь кликает на аннотацию (обводку) на изображении:
- В инпут добавляется визуальный маркер (@кольцо)
- Маркер имеет цвет соответствующий типу аннотации
- Можно добавить несколько маркеров для разных областей

### 2. Визуальный маркер в инпуте
Маркер показывает:
- **@** символ в цветном кольце
- **Название аннотации** (label)
- **Имя изображения** для контекста
- **Кнопку удаления** (X)

### 3. Отправка сообщения
При отправке:
- Маркеры сохраняются в сообщении как `annotationRefs`
- Отображаются как красивые интерактивные кнопки
- Gemini получает текстовое описание областей

### 4. Клик на маркер в сообщении
При клике на маркер в отправленном сообщении:
- **Открывается lightbox** с изображением
- **Показывается только выбранная аннотация** (та, на которую кликнули)
- Можно увеличивать, перемещать изображение
- Видна обводка и метка области

## Визуальное отображение

### В инпуте (перед отправкой):
```
┌─────────────────────────────────────┐
│ [@] Логотип · photo.jpg        [X]  │ ← Желтый маркер
│ [@] Кнопка входа · screen.png  [X]  │ ← Синий маркер
└─────────────────────────────────────┘
```

### В сообщении (после отправки):
```
┌─────────────────────────────────────┐
│ [@] Логотип                         │ ← Кликабельная кнопка
│     photo.jpg                    👁️ │   с иконкой глаза при hover
│                                     │
│ [@] Кнопка входа                    │
│     screen.png                   👁️ │
└─────────────────────────────────────┘
```

### При клике → Lightbox:
- Открывается изображение
- Показана только выбранная область с обводкой
- Можно зумить и перемещать

| Тип | Цвет | Использование |
|-----|------|---------------|
| `highlight` | 🟡 Желтый (#FBBF24) | Важные элементы |

## Пример использования

### Сценарий: Анализ UI элементов

1. **Пользователь загружает скриншот интерфейса**
2. **Gemini анализирует** и создает аннотации:
   - "Логотип" (highlight, желтый)
   - "Кнопка входа" (pointer, синий)
   - "Форма регистрации" (info, фиолетовый)

3. **Пользователь кликает на "Кнопка входа"**
   - В инпуте появляется синий маркер: `[@] Кнопка входа · screenshot.png`

4. **Пользователь пишет**: "Какой размер и стиль?"

5. **Отправка**:
   - В сообщении красивая синяя кнопка с маркером
   - Gemini получает: "Какой размер и стиль?\n\nСсылаюсь на: @[Кнопка входа] (на изображении "screenshot.png", область: 45.2%-60.1% to 55.8%-65.3%)"

6. **Пользователь кликает на маркер в сообщении**:
   - Открывается lightbox
   - Показано изображение с синей обводкой вокруг кнопки
   - Можно увеличить для детального просмотра
