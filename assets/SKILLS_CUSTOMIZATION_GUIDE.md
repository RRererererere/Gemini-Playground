# Руководство по кастомизации скилов

## Обзор

Система кастомизации скилов позволяет пользователям полностью настраивать поведение каждого скилла под свои нужды. Вы можете изменять:

- **Описание скилла** — как скилл представлен в маркете
- **Системный промпт** — инструкции для AI о том, как использовать скилл
- **Описания инструментов** — как AI понимает каждый tool в скилле

Все настройки сохраняются локально и могут быть экспортированы/импортированы.

## Как использовать

### Открытие настроек скилла

1. Откройте **Маркет скиллов** (кнопка ✨ Скиллы)
2. Найдите нужный скилл (установленный или в каталоге)
3. Нажмите кнопку **⚙️ Настройки** (иконка шестерёнки)

### Вкладки настроек

#### ⚙️ Настройки
Стандартные параметры скилла (API ключи, конфигурация). Зависит от конкретного скилла.

#### 📝 Промпт
Здесь вы можете настроить:

**Описание скилла**
- Показывается в маркете и влияет на то, как пользователи понимают назначение скилла
- Оставьте пустым для использования дефолтного описания
- Пример: вместо "Точные математические вычисления" можно написать "Калькулятор для сложных расчётов с поддержкой формул"

**Системный промпт**
- Инструкции для AI о том, когда и как использовать скилл
- Добавляется в system prompt при каждом запросе к API
- Переопределяет встроенный `onSystemPrompt` скилла
- Пример для скилла Notes:
  ```
  У пользователя есть система заметок. Ты можешь:
  - Создавать заметки через create_note (важная информация, задачи, идеи)
  - Читать все заметки через list_notes
  - Удалять заметки через delete_note
  
  ВАЖНО: Всегда предлагай сохранить важную информацию в заметки.
  ```

#### 🔧 Инструменты
Настройка описаний каждого tool в скилле:

- Видите список всех инструментов скилла
- Для каждого показано дефолтное описание
- Можете написать своё описание, которое AI будет видеть
- Это влияет на то, как AI понимает когда и как вызывать инструмент

**Пример для calculator.calculate:**

Дефолтное:
```
Вычисляет математическое выражение. Используй для точных расчётов: 
арифметика, проценты, формулы. НЕ считай в уме — используй этот инструмент.
```

Кастомное (более строгое):
```
ОБЯЗАТЕЛЬНО используй этот инструмент для ЛЮБЫХ вычислений, даже простых.
НИКОГДА не считай в уме. Примеры: 2+2, 15*8, (100-20)/4, 2**10.
Всегда вызывай calculate перед тем как дать числовой ответ.
```

### Сохранение и сброс

**Сохранить кастомизацию** — применяет все изменения
**Сбросить к умолчаниям** — удаляет все кастомные настройки, возвращает дефолтные значения

## Экспорт и импорт настроек

### Экспорт

1. Откройте Маркет скиллов
2. Нажмите кнопку **📥 Download** в правом верхнем углу
3. Скачается JSON файл со всеми настройками всех скиллов

Файл содержит:
- Список установленных скиллов
- Статус (включён/выключен)
- Конфигурацию (API ключи и т.п.)
- Кастомизацию (описания, промпты, tool descriptions)

### Импорт

1. Откройте Маркет скиллов
2. Нажмите кнопку **📤 Upload** в правом верхнем углу
3. Выберите JSON файл с настройками
4. Система автоматически:
   - Установит скиллы если их нет
   - Применит все настройки и кастомизацию
   - Покажет результат (сколько успешно, сколько ошибок)

**Применение:** 
- Перенос настроек между устройствами
- Бэкап конфигурации
- Шаринг настроек с другими пользователями
- Быстрая настройка новой установки

## Формат экспорта

```json
{
  "version": "1.0.0",
  "exportedAt": 1234567890,
  "skills": [
    {
      "id": "calculator",
      "enabled": true,
      "config": {},
      "customization": {
        "customDescription": "Мой кастомный калькулятор",
        "customSystemPrompt": "Всегда используй калькулятор для вычислений",
        "customToolDescriptions": {
          "calculate": "ОБЯЗАТЕЛЬНО вызывай для любых расчётов"
        }
      }
    }
  ]
}
```

## Примеры использования

### Пример 1: Строгий калькулятор

**Проблема:** AI иногда считает в уме вместо использования калькулятора

**Решение:**
1. Откройте настройки скилла Calculator
2. Вкладка "Промпт" → Системный промпт:
   ```
   КРИТИЧЕСКИ ВАЖНО: ВСЕГДА используй инструмент calculate для ЛЮБЫХ 
   математических операций. НИКОГДА не считай в уме, даже для простых 
   примеров типа 2+2. Это обязательное требование.
   ```
3. Вкладка "Инструменты" → calculate:
   ```
   ОБЯЗАТЕЛЬНЫЙ инструмент для ВСЕХ вычислений без исключений.
   Вызывай ПЕРЕД тем как дать любой числовой ответ.
   Примеры: 2+2, 10*5, (100-20)/4, 2**10, 15% от 200
   ```

### Пример 2: Проактивные заметки

**Проблема:** AI редко предлагает сохранить информацию в заметки

**Решение:**
1. Откройте настройки скилла Quick Notes
2. Вкладка "Промпт" → Системный промпт:
   ```
   У пользователя есть система заметок. Текущие заметки:
   [список будет добавлен автоматически]
   
   ВАЖНО: Активно предлагай сохранять в заметки:
   - Важные факты и данные
   - Задачи и напоминания
   - Идеи и планы
   - Полезные ссылки
   
   Спрашивай: "Сохранить это в заметки?" когда обсуждается что-то важное.
   ```

### Пример 3: Специализированный URL Reader

**Проблема:** Нужно читать только технические статьи и документацию

**Решение:**
1. Откройте настройки скилла URL Reader
2. Вкладка "Промпт" → Системный промпт:
   ```
   Используй read_url для чтения технической документации и статей.
   Фокусируйся на: API docs, tutorials, technical specs, GitHub README.
   После чтения всегда выделяй ключевые технические детали.
   ```
3. Вкладка "Инструменты" → read_url:
   ```
   Читает техническую документацию и статьи. 
   Используй когда нужно проверить API, изучить библиотеку, 
   прочитать туториал или техническую спецификацию.
   Всегда извлекай примеры кода и важные технические детали.
   ```

## Технические детали

### Приоритет настроек

1. **Кастомное описание** переопределяет `skill.description`
2. **Кастомный системный промпт** переопределяет `skill.onSystemPrompt()`
3. **Кастомные описания инструментов** переопределяют `tool.description`

### Хранение

Все настройки хранятся в `localStorage`:
- Ключ: `skills_registry`
- Структура: `InstalledSkillRecord` с полем `customization`

### API

```typescript
// Получить кастомизацию
import { getSkillCustomization } from '@/lib/skills/registry';
const custom = getSkillCustomization('calculator');

// Сохранить кастомизацию
import { saveSkillCustomization } from '@/lib/skills/registry';
saveSkillCustomization('calculator', {
  customDescription: 'Мой калькулятор',
  customSystemPrompt: 'Всегда используй для расчётов',
  customToolDescriptions: {
    calculate: 'Обязательный инструмент'
  }
});

// Сбросить к умолчаниям
import { resetSkillCustomization } from '@/lib/skills/registry';
resetSkillCustomization('calculator');

// Экспорт/импорт
import { exportSkillsSettings, importSkillsSettings } from '@/lib/skills/registry';
const data = exportSkillsSettings();
const result = importSkillsSettings(data);
```

## Best Practices

### Системные промпты

✅ **Хорошо:**
- Конкретные инструкции
- Примеры использования
- Чёткие правила (ВСЕГДА/НИКОГДА)
- Контекст когда использовать

❌ **Плохо:**
- Расплывчатые формулировки
- Слишком длинные тексты
- Противоречивые инструкции
- Дублирование дефолтного промпта

### Описания инструментов

✅ **Хорошо:**
- Когда вызывать инструмент
- Примеры параметров
- Ожидаемый результат
- Ограничения и edge cases

❌ **Плохо:**
- Только название без контекста
- Технический жаргон без объяснений
- Отсутствие примеров
- Неполная информация о параметрах

### Организация

- Экспортируйте настройки регулярно (бэкап)
- Тестируйте изменения на простых примерах
- Документируйте свои кастомизации
- Делитесь удачными настройками с командой

## Troubleshooting

**AI не использует кастомные инструкции**
- Проверьте что скилл включён
- Убедитесь что сохранили изменения
- Перезагрузите страницу
- Проверьте что промпт не пустой

**Импорт не работает**
- Проверьте формат JSON файла
- Убедитесь что версия совместима
- Проверьте консоль браузера на ошибки

**Настройки не сохраняются**
- Проверьте что localStorage не заблокирован
- Очистите кэш браузера
- Попробуйте в режиме инкогнито
